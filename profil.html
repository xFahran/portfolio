Profil

Un développeur qui pense architecture, sécurité et maintenabilité.

Je viens du développement web et mobile, avec une approche très orientée backend, qualité logicielle et sécurité. Je sais produire une interface, mais je ne m'arrête pas au joli bouton : je m'intéresse surtout à ce qu'il déclenche, comment les données circulent, comment les erreurs sont gérées et comment éviter qu'une application devienne une dette technique ambulante.

Méthode

Ce que j'apporte

Une vision pragmatique du développement : comprendre le besoin, choisir une architecture lisible, sécuriser les entrées, structurer les données et livrer un code exploitable.

Positionnement

Full stack, avec un vrai tropisme backend

Je peux intervenir sur une API Node.js, une application WPF, une interface EJS, une base PostgreSQL ou MariaDB, un environnement Docker, ou une application C++ avec Qt.

01

Code défensif

Validation des entrées, gestion propre des erreurs, droits d'accès et logique claire. Le minimum vital, bizarrement encore optionnel dans trop de projets.

02

Architecture lisible

Des dossiers cohérents, des responsabilités séparées, des services compréhensibles et un code qui ne donne pas envie de fermer le PC.

03

Qualité logicielle

Une logique issue du test logiciel : cas d'usage, anomalies, couverture, critères d'acceptation et maintenabilité.
