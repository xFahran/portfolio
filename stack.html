Stack technique

Les outils que j'utilise vraiment.

Une stack construite pour produire des applications solides : backend clair, bases fiables, interfaces propres, environnements maîtrisés et qualité logicielle en fil rouge.

TSLangage

TypeScript

Je l'utilise pour fiabiliser le JavaScript, rendre les contrats de données plus explicites et limiter les erreurs bêtes qui apparaissent quand un projet grossit.

NBackend

Node.js

Mon environnement principal côté backend. Je m'en sers pour construire des API, gérer la logique métier, structurer les services et connecter proprement l'application à la base de données.

EXAPI

Express

Framework simple et efficace pour créer des routes, middlewares, contrôleurs et API REST. Je l'utilise avec une structure claire pour éviter le fichier serveur de 900 lignes.

SQORM

Sequelize

ORM que j'utilise pour structurer les modèles, sécuriser les interactions avec la base, gérer les associations et éviter les requêtes SQL bricolées à la main quand ce n'est pas nécessaire.

PGData

PostgreSQL

Base relationnelle solide pour les projets nécessitant cohérence, transactions et structure sérieuse des données. Je l'associe souvent à Sequelize et Docker.

DBData

MariaDB

Je l'utilise pour des projets web classiques ou internes, notamment quand l'environnement attend une base compatible MySQL avec une configuration simple.

HFront

HTML

Base de toute interface web propre. Je l'utilise avec une attention particulière à la sémantique, l'accessibilité et la structure.

CUI

CSS

Je m'en sers pour produire des interfaces modernes, responsive et animées sans transformer le projet en carnaval illisible.

JSFront

JavaScript

Indispensable pour l'interactivité côté client. Je l'utilise surtout quand le besoin ne justifie pas une surcouche inutile.

ETemplate

EJS

Moteur de templates utile pour générer des vues côté serveur avec Node.js, notamment sur des sites simples, rapides et maintenables.

BUI kit

Bootstrap

Pratique pour accélérer la mise en page, prototyper vite et garder une base responsive cohérente quand le projet n'exige pas un design entièrement custom.

PCMS

PHP

Je l'utilise principalement dans le contexte CMS, notamment WordPress et PrestaShop, pour comprendre, adapter ou maintenir des fonctionnalités existantes.

WCMS

WordPress

CMS utile pour des sites vitrines, contenus éditoriaux et projets clients où l'autonomie de gestion est importante.

PSCommerce

PrestaShop

Solution e-commerce que j'aborde côté intégration, personnalisation, compréhension métier et architecture boutique.

C#Desktop

C# / WPF

Stack desktop pour créer des applications Windows structurées, avec séparation interface/logique et composants adaptés aux besoins métier.

C++Native

C++ / Qt

Pour les applications natives performantes, avec interfaces desktop et connexion à des bases comme PostgreSQL.

DDevOps

Docker

Je l'utilise pour isoler les environnements, stabiliser les dépendances et rendre les projets plus simples à déployer ou reprendre.

GVersioning

Git

Indispensable pour versionner proprement, travailler en branches, sécuriser les évolutions et garder un historique exploitable.

LXSystème

Linux

Base solide pour le serveur, les environnements de développement, les permissions, le shell et la compréhension de ce qui tourne vraiment sous le capot.

FDesign

Figma

Je l'utilise pour maquetter, clarifier les parcours utilisateurs et éviter de coder une interface sans vision.

UConception

UML / MCD / MLD

Outils de conception pour modéliser avant de coder, poser les relations et éviter de découvrir l'architecture au milieu du développement.

SSécurité

Sécurité

Approche défensive : validation des entrées, gestion des erreurs, contrôle d'accès, hygiène des dépendances et attention portée aux risques OWASP.

QAISTQB

Tests logiciel

Grâce à mes certifications ISTQB, je raisonne aussi en stratégie de test, cas de test, critères d'acceptation, anomalies et qualité livrable.
